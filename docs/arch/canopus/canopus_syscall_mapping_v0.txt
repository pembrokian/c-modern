Canopus sys v0 — Kernel Syscall Mapping

Status: companion to sys v0
Scope: exact kernel syscall mapping for the sys interface
Audience: kernel + runtime implementers

⸻

1. Purpose

This note maps each operation in sys v0 to an exact kernel syscall contract.

The goal is to keep the boundary one-to-one:
	•	each sys operation maps to one kernel syscall
	•	the syscall ABI uses status returns plus caller-provided output storage
	•	no syscall performs hidden namespace lookup or runtime policy

This document is intentionally kernel-facing.
It does not prescribe the final Modern C wrapper syntax above the syscall layer.

⸻

2. Shared ABI Rules

2.1 Status Return

Every fallible syscall returns Status.

enum Status {
    Ok,
    Invalid,
    PermissionDenied,
    NotFound,
    Exhausted,
    TooLarge,
    WouldBlock,
    Unknown,
}

2.2 Handle Representation

Kernel syscalls operate on process-local handle values.
At the syscall ABI these may be represented as machine-sized integers.
At the language boundary they remain opaque handles.

2.3 Output Storage

Whenever a syscall needs to produce data, the caller supplies writable output storage.
The kernel never allocates output buffers on behalf of user code.

2.4 Copy And Move Semantics

Byte buffers are copied.
Capability transfers are moves on success and no-ops on failure.

⸻

3. Exact Mapping

3.1 Endpoint Creation

sys operation:
	•	endpoint_create(out: *Endpoint) -> Status

kernel syscall:
	•	sys_endpoint_create(out_handle: *Handle) -> Status

kernel effect:
	•	allocates a new endpoint object
	•	installs a handle into the calling process handle table
	•	writes that handle to out_handle

failure cases:
	•	Invalid if out_handle is not writable
	•	Exhausted if the kernel cannot allocate the endpoint or handle-table slot

⸻

3.2 Message Send

sys operation:
	•	send(ep: Endpoint, msg: SendMessage) -> Status

kernel syscall:
	•	sys_ipc_send(
		endpoint: Handle,
		data_ptr: *u8,
		data_len: u64,
		caps_ptr: *Handle,
		cap_count: u64
	) -> Status

kernel effect:
	•	validates endpoint handle and each transferred capability handle
	•	copies byte payload into kernel message storage
	•	queues one message containing bytes plus capability payload
	•	on success, removes transferred capabilities from the sender handle table

failure cases:
	•	PermissionDenied if endpoint or transferred handles are not valid for this process
	•	TooLarge if byte or capability payload exceeds kernel limits
	•	WouldBlock if v0 chooses bounded queues and the endpoint cannot currently accept the message
	•	Exhausted if kernel message storage cannot be reserved

⸻

3.3 Message Receive

sys operation:
	•	recv(ep: Endpoint, msg: *RecvMessage) -> Status

kernel syscall:
	•	sys_ipc_recv(
		endpoint: Handle,
		data_ptr: *u8,
		data_cap: u64,
		caps_ptr: *Handle,
		caps_cap: u64,
		out_recv_len: *u64,
		out_recv_caps: *u64
	) -> Status

kernel effect:
	•	reads one queued message from the endpoint
	•	copies message bytes into caller storage
	•	installs transferred capabilities into the receiver handle table
	•	writes byte count to out_recv_len
	•	writes capability count to out_recv_caps

failure cases:
	•	PermissionDenied if endpoint is not a valid receiving handle for this process
	•	TooLarge if caller storage cannot hold the entire message; the message remains queued
	•	WouldBlock if no message is available and the endpoint is configured for non-ready blocking semantics
	•	Invalid if any output pointer is not writable

⸻

3.4 Process Spawn

sys operation:
	•	spawn(spec: ProcSpec, out: *Process) -> Status

kernel syscall:
	•	sys_proc_spawn(
		image_cap: Handle,
		argv_ptr: *ArgSpan,
		argv_count: u64,
		caps_ptr: *Handle,
		cap_count: u64,
		out_process: *Handle
	) -> Status

where:

ArgSpan {
    ptr: *u8
    len: u64
}

kernel effect:
	•	validates image_cap as a launchable program capability
	•	creates a new process
	•	installs the supplied capability handles into the child process handle table
	•	writes the new process handle to out_process

failure cases:
	•	PermissionDenied if image_cap does not carry spawn authority or any transferred cap is invalid
	•	NotFound if image_cap names a stale or unloaded executable object
	•	Exhausted if the kernel cannot allocate the process or associated resources
	•	Invalid if argv storage is malformed or an output pointer is not writable

notes:
	•	the kernel does not resolve file paths here
	•	image_cap is produced by a loader or parent process outside this syscall

⸻

3.5 Process Wait

sys operation:
	•	wait(p: Process, out_exit_code: *i32) -> Status

kernel syscall:
	•	sys_proc_wait(process: Handle, out_exit_code: *i32) -> Status

kernel effect:
	•	blocks until the target process exits
	•	writes the process exit code to out_exit_code

failure cases:
	•	PermissionDenied if the calling process lacks authority to observe the target process
	•	Invalid if out_exit_code is not writable

⸻

3.6 Process Kill

sys operation:
	•	kill(p: Process) -> Status

kernel syscall:
	•	sys_proc_kill(process: Handle) -> Status

kernel effect:
	•	requests termination of the target process

failure cases:
	•	PermissionDenied if the calling process lacks kill authority for the target process
	•	NotFound if the process has already fully exited and been reaped

⸻

3.7 Time Read

sys operation:
	•	now() -> Time

kernel syscall:
	•	sys_time_now() -> u64

kernel effect:
	•	returns the current monotonic system time value

notes:
	•	v0 keeps this infallible
	•	the unit must be defined consistently across kernel and user space; milliseconds are acceptable for bring-up, nanoseconds are preferable if cheap

⸻

3.8 Sleep

sys operation:
	•	sleep(ms: u64) -> Status

kernel syscall:
	•	sys_time_sleep(duration_ms: u64) -> Status

kernel effect:
	•	blocks the calling process or thread of execution for at least the requested duration

failure cases:
	•	Invalid only for nonsensical platform-specific ranges if the kernel imposes one

⸻

4. Deliberate Non-Mapping

sys v0 has no alloc/free mapping.
Heap allocation is not a kernel syscall concern in this interface revision.

⸻

5. Recommended Kernel Surface

The minimal kernel syscall set for sys v0 is therefore:
	•	sys_endpoint_create
	•	sys_ipc_send
	•	sys_ipc_recv
	•	sys_proc_spawn
	•	sys_proc_wait
	•	sys_proc_kill
	•	sys_time_now
	•	sys_time_sleep

This is small enough for bring-up while still making explicit capability transfer a first-class mechanism.
