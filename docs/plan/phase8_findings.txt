Phase 8 -- Findings Log
=======================

Purpose
-------
- Record what each admitted pressure program exposed while it was being implemented.
- Keep implementation fixes, bootstrap gaps, and post-v0.2 design questions separate.

Taxonomy
--------
- implementation bug: the intended repository-owned capability exists, but the compiler, stdlib, runtime, or proof harness behaves incorrectly.
- bootstrap capability gap: the admitted repository slice does not yet contain a needed capability, but the gap is still within the current bootstrap direction.
- stdlib or runtime gap: the compiler works, but the library or hosted runtime boundary is too thin for the admitted pressure program.
- tooling gap: build, run, test, project, or artifact wiring is insufficient even though language and library support are otherwise adequate.
- spec-level follow-up: repeated pressure exposed a language or build-model awkwardness that should not be reopened until multiple programs point at the same problem.

Decision rule
-------------
- Fix implementation, stdlib, runtime, or tooling truth first.
- Reopen language or build-model questions only after repeated multi-program pressure.
