Phase 8 -- Admission Matrix
===========================

Purpose
-------
- Turn Phase 8 program admission into repository truth instead of prose inference.
- Record, per pressure program, what is being stressed, how it is proven, and whether it is implemented, blocked, or deferred.
- Phase 14A refreshes this matrix against `docs/arch/canonical_programs.txt`, the
	landed first-pass Phase 8 slice, and older lower-layer canonical artifacts
	that already exist in the repository.

Status model
------------
- implemented: the repository contains the program and deterministic proof.
- in-progress: the repository contains part of the program or proof, but the pressure artifact is not yet complete.
- blocked: the program is admitted, but a named enabling gap prevents honest proof inside the admitted bootstrap slice.
- deferred: the program remains part of the long-term pressure set, but it is outside the first-pass Phase 8 closure for this repository.

Proof path defaults
-------------------
- Use direct-source proof for single-file canonical programs whose pressure value does not depend on explicit project layout.
- Use dedicated build.toml fixtures for multi-module programs and for any proof that is meant to validate the explicit Phase 7 project workflow.
- Ordinary tests for project-style pressure programs must import the real project modules through configured roots.

Canonical status notes
----------------------
- implemented: the repository contains the program and deterministic proof at
	the intended Phase 8 pressure layer.
- legacy lower-layer proof: the repository contains a real canonical example or
	fixture, but it is currently proven only at a lower layer such as MIR and is
	not yet counted as a full admitted Phase 8 executable or project proof.
- blocked: the program is meaningful and remains in scope, but a named
	enabling gap prevents honest admission on the current hosted slice.
- deferred: the program remains part of the long-term pressure set, but the
	repository is not yet admitting it beyond explicit backlog and owner naming.
