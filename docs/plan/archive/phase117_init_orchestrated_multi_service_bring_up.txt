Phase 117 -- Init-Orchestrated Multi-Service Bring-Up
=====================================================

Status
------
IMPLEMENTED

Purpose
-------
Continue from the completed Phase 116 MMU activation barrier follow-through by
making the next running-system claim explicit: init now orchestrates one
bounded multi-service set over the landed service owners instead of the
repository treating those service proofs only as separate phase-local slices.

This document is both the Phase 117 execution note and its closeout.
It does not claim general service discovery, dynamic loading, a broader
runtime framework, namespace policy, or a new compiler wave.
It records one narrow running-system follow-through inside the already
admitted first running Canopus kernel slice.

- Phase 105 through Phase 107 already made three bounded service-shaped

Problem being solved
--------------------
- The speculative post-Phase-116 roadmap reserved Phase 117 for one bounded
  init-orchestrated multi-service bring-up if the landed service owners and
  running-kernel slice were already strong enough.
- The real kernel artifact already launched and observed the bounded log,
  echo, and transfer service flows through explicit program capabilities,
  endpoint authority, wait handles, and user-space policy.
- But the repository still published that slice as if no broader multi-service
  bring-up had landed, which left the implementation stronger than the claim
  and blurred the next honest boundary between a bounded running-system proof
  and a speculative service framework.

Core rule
---------
- Preserve the current kernel behavior exactly.
- Keep service policy in `init` or user-space-facing orchestration rather than
  moving it into a new kernel framework.
- Keep the compiler, sema, MIR, backend, ABI, and `hal` surfaces closed.
- Do not widen into dynamic loading, service discovery, a namespace model, or
  a general running-system framework.
- Prove the boundary mechanically with one aggregate audit over the real
  kernel artifact and one focused freestanding regression.
