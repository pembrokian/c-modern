Problem being solved
--------------------
- The repository already had one explicit service-death observation answer,

  a hidden recovery story.
- Without this step, later failure, IPC-composition, or restart work could
  blur whether the current model already keeps one dead-service path explicit
  while another bounded service path still survives.
- The repository needed one small phase that classifies one dead log-service
  path together with one fresh echo-service reply path through the existing
  bootstrap-service, fixed-directory, shared-control-endpoint, wait-handle,
  and debug owners rather than through a new recovery or rebinding layer.

- Keep the compiler, sema, MIR, backend, ABI, and `hal` surfaces closed.

- Prove one bounded service death remains partial because a second bounded
