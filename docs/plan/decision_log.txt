Modern C - Decision Log

Purpose

This file records implementation choices where the team intentionally chose a limited or provisional solution because a fuller solution was blocked, too broad, or not yet justified.

Use this file for:
- limited solutions chosen during implementation
- cases where a deeper fix was deferred on purpose
- compiler-layer workarounds that should be revisited later
- decisions that are not language-spec questions but still matter for future cleanup

Do not use this file for:
- unresolved questions that belong in docs/arch/open_decisions.txt
- durable settled design rationale that belongs in docs/arch/key_decisions.txt
- ordinary progress notes or implementation summaries

Suggested entry shape

- date: YYYY-MM-DD
- area: parser | sema | mir | codegen | runtime | stdlib | tooling | docs
- context: what was being implemented
- limiter: what prevented the fuller solution
- decision: what limited solution was chosen
- tradeoff: what was accepted or sacrificed
- revisit trigger: what should cause this to be reopened
- reference: related phase doc, backlog item, or code location
